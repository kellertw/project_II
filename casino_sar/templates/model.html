Utilizing sqlite database queries to construct Python DataFrames in order to analyze connections between SARS reporting and other variables within dataset.

Our group was able to convert the "Industry" column into categorical data which was necessary for our modeling. We created a dictionary of dictionaries with the column objects inside the nested dictionary assigned to ordinal values. Afterwards, we cleaned the data by running the .replace() function.

The model we worked with with utilizes components of both supervised and unsupervised machine learning. First, we established a pipeline that scales the model, reduces dimensionality, and provides the test sample of the model all within the same cell. Then we called the dependencies and imports. RunTest() is a function that tests our model using the pipeline's preprocessing output as its parameters. Once RunTest() completes, the last step is to loop through each of the libraries we called prior to setting up the pipeline and RunTest(). All dependencies and libraries containing the statistical tests we used are declared before the function, RunTest(), is executed. The cell then returns the model's precision as well as the model that yielded the highest precision score and the number of features it looped through to get those results.

In order to run our first machine learning test, we dropped columns containing string values keeping only "Count", "Year", and "Industry" and converted the "Count" column datatypes. to float using .to_numeric().
